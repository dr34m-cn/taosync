## 🚀 特性

* 新增简体中文与 English 双语支持：前端界面、Element Plus 组件、后端接口提示、启动日志、任务状态和通知内容统一国际化；支持自动识别、手动切换、本地记忆和中文回退
* 新增亮色/暗色主题切换，完整适配登录、导航、作业、任务、引擎、通知和设置等页面，并记忆用户选择 [#108](https://github.com/dr34m-cn/taosync/issues/108) [#186](https://github.com/dr34m-cn/taosync/pull/186)
* 新增 Android 客户端（arm64-v8a、armeabi-v7a）：业务服务运行在独立服务进程，提供本机启动日志页、局域网访问、电池优化白名单引导和自适应应用图标 [#168](https://github.com/dr34m-cn/taosync/issues/168) [#225](https://github.com/dr34m-cn/taosync/pull/225)
* 通知方式新增企业微信应用消息和 Lark 群机器人，支持配置校验、测试发送与“忽略无同步”；企业微信应用 Secret 改为密码方式展示 [#183](https://github.com/dr34m-cn/taosync/pull/183) [#188](https://github.com/dr34m-cn/taosync/pull/188)
* 定时作业新增 15 分钟失效宽限期，短暂休眠或调度错过后仍允许补执行 [#145](https://github.com/dr34m-cn/taosync/issues/145) [#152](https://github.com/dr34m-cn/taosync/issues/152)
* 新增 Linux StaticX 独立可执行文件构建，覆盖 amd64、arm64 和 arm32，为缺少 musl/动态库的 ARM 与 Linux 环境提供可移植构建 [#30](https://github.com/dr34m-cn/taosync/issues/30) [#52](https://github.com/dr34m-cn/taosync/issues/52) [#111](https://github.com/dr34m-cn/taosync/issues/111) [#202](https://github.com/dr34m-cn/taosync/issues/202)

## 🎨 优化

* 前端由 Vue 2、Vue CLI、Element UI 和 Vuex 重构为 Vue 3、Vite、Element Plus 和 Pinia，重新组织路由、状态、API 与通用组件
* 完整重做移动端交互：作业管理、任务历史和任务项详情均使用独立移动页面，增加底部导航、抽屉编辑、卡片列表、移动筛选和分页；桌面端导航支持折叠
* 优化网络与会话处理：请求携带当前语言，统一本地化错误提示，会话失效时清理旧 Cookie 并根据当前页面决定跳转或保留
* 后端语言文案由代码内硬编码迁移到 YAML 语言包，增加语言标识兼容、按请求的语言上下文与缺失文案回退
* 重整自动构建流程：前端统一构建后供 Docker、Windows、macOS、StaticX 和 Android 复用，Android/StaticX 构建合并进主工作流，macOS Intel 构建环境更新至 macOS 15 [#189](https://github.com/dr34m-cn/taosync/issues/189) [#190](https://github.com/dr34m-cn/taosync/pull/190)

## 🐞 修复

* 修复 PyInstaller 打包后语言包路径无法正确定位，以及 `charset-normalizer` 依赖和资源未被完整收集导致的启动或请求异常
* 修复 Docker 构建阶段未显式安装项目依赖的问题，并在 Android APK 缺失时阻止产物上传与 Release 发布
* 配置/语言文件读写改为 UTF-8，API 响应明确返回 UTF-8 JSON，避免中文配置或提示乱码
